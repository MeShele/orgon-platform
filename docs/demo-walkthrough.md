# ORGON — Demo Walkthrough Script

**Цель документа:** runbook для презентующего ORGON клиенту B2B (криптообменник, брокер, банк).
**Длительность:** 12–15 минут.
**Площадка:** `https://orgon.asystem.ai` (production). Демо-кредсы в шаге 3.
**Главный wow-момент:** интерактивный архитектурный симулятор (`/demo/architecture`, шаг 2) — показать его обязательно даже если ужимаемся по времени.

- [ ] Разрешение экрана: 1440×900 минимум; для мобильной части (опциональной) — заранее показать в DevTools.

**URL:** `https://orgon.asystem.ai/`

**Если что-то не грузится:** обновить страницу, проверить `https://orgon.asystem.ai/api/health`. Если 5xx — позвать инженера, **перейти сразу на шаг 2** (симулятор не зависит от бэкенда).

## Шаг 2 — Архитектурный симулятор (~3–4 мин) **[main wow]**

**URL:** `https://orgon.asystem.ai/demo/architecture`

**Ожидаемый визуал:** sticky-граф React Flow слева, 6 сценариев справа. Узлы (Customer / API / RBAC / RLS / Validators / Safina / Compliance / Audit) — реальные компоненты системы. Лейблы вынесены наверх над графом в виде колонок.

**Демо-сценарии — крутить минимум первые 2:**

1. **Withdrawal happy path** — стандартный поток вывода. Показать как запрос проходит через RBAC → RLS → Validators → Safina → блокчейн.
2. **Sanctions block** — как Compliance-узел блокирует транзакцию по совпадению с санкционным списком. **Это убедительный сценарий — клиенты слышат ровно про это от регуляторов.**
3. (опционально) **Replay attack** — HMAC-нонс отбивает повтор. Технические клиенты ценят.
4. (опционально) **RLS isolation** — две организации параллельно, никто никого не видит. Multi-tenancy как заявлена.
5. (опционально) **Night window** — политика "не выводить ночью" блокирует платёж.
6. (опционально) **Webhook retry** — exponential backoff, идемпотентный ключ.

**Ключевые тезисы:**
- "Все узлы и связи — реальные компоненты кода." Симулятор — не маркетинг-картинка. Можно кликнуть на любой блок и увидеть детали.
- "M-of-N подписи у нас не на бумаге — у каждой транзакции реальная цепочка signature_history с триггером append-only на уровне БД."

**Если симулятор не грузится:** F5; если не помогло — открыть `/features` и говорить тезисы оттуда.

**URL:** клик "Кошельки" в сайдбаре или `/wallets`

**Ожидаемый визуал:** таблица из 15 кошельков. **Display name теперь читабельный:** `BSC · 11FCEC93…F5` или подобный (раньше был полный 32-символьный hex). Network-бейдж справа от имени. Hover на строке показывает полный UNID в tooltip.

**Кликнуть в любой кошелёк** — показать детальную страницу. Адрес, токены, история операций. Это **живой сетевой объект**, не мок.

## Доказательство что Safina реально работает

Если CTO/security клиента спрашивает "это не моки?" — **открой терминал**:

```
./scripts/safina-proof.sh
```

За 10 секунд скрипт делает 5 проверок и выводит зелёные ✓:

1. JWT-логин под demo-admin
2. `/api/health/safina` — `safina_reachable: true` (живой TLS-handshake с `my.safina.pro/ece/` на каждом запросе)
3. `/api/health/detailed` — `services.safina_api.status: healthy` + статусы Postgres / cache
4. `/api/wallets` — 15 кошельков с **каноничными Safina-UNID** (16-байт hex, формат Safina) — stub их не знал бы
5. `/api/transactions` — реальная tx с `safina_id: 230` (FK на primary-key в БД Safina) и **проверяемым на блокчейне Tron-адресом** (tronscan.org)
6. `/api/networks` — 7 сетей с реальными URL block-explorer'ов из Safina (включая `ORGON` и `ORGON TestNet`)

**Что доказывает каждое:**

| Сигнал | Почему stub так не сделает |
|---|---|
| `safina_reachable: true` | Это live ECDSA-подписанный запрос к Safina API. Stub вернул бы `false` или вообще не отвечал бы. |
| `safina_id: 230` | Это FK на primary-key Safina. Stub не угадал бы их нумерацию. |
| `UNID = 11FCEC937030E81C45258DE8002BE6F5` | Каноничный 16-байт hex-формат Safina (timestamp+random). Stub генерил бы UUID4. |
| `to_addr = TRx6xXChS5sXz3mpvLSNfKuL6w3PBdMZzL` | Реальный Tron-адрес — открой на tronscan, увидишь блокчейн. |
| `block_explorer = https://tronscan.org/...` | Метаданные сетей пришли из Safina, не захардкожены в ORGON. |

**Дополнительно** — в `/api/docs` (Swagger UI) есть **193 операции** на 166 путях. Клиент может сам авторизоваться и потыкать "Try it out".

## Что показывать НЕ нужно

- `/compliance`, `/documents`, `/users`, `/settings` (общий) — сидят в группе "Дорожная карта" с бейджем "Скоро". Если клиент кликнет — не страшно, страница откроется с честной плашкой "В разработке". **Не углубляться, идти дальше.**
- DevTools / network tab / любые JSON-ответы. Если очень просят — показать только `/api/health` (publicly accessible).
- Логи из контейнеров — это шум.
- Stripe billing — endpoint возвращает 503 пока не настроены price IDs. Не открывать на демо.

## Fallback-плейбук (когда что-то ломается)

| Симптом | Что делать |
|---|---|
| `orgon.asystem.ai` 5xx | Пингануть `https://orgon.asystem.ai/api/health`. Если 5xx — простите за глюк, перейти на симулятор (`/demo/architecture` — статичный, не зависит от backend). |
| Логин не работает | Проверить кредсы (`demo-admin@orgon.io / demo2026`). Если "Invalid credentials" — попросить тех-команду проверить (могло сменится). Перейти к симулятору. |
| Recent activity пусто, но клиент хочет видеть транзакции | Сказать: "Это test-аккаунт без депозитов" + перейти на симулятор для **виртуальных** транзакций. |
| Симулятор тормозит на мобайле | Открыть на ноутбуке. Симулятор оптимизирован для desktop/tablet (≥768px). |
| WebSocket disconnected на dashboard | F5; если воспроизводится — перейти к статическим страницам (`/wallets`, `/audit`). WebSocket не критичен для демо. |
